# Provenance Record — Ontology Documentation

## Definition

A trace of data origin, transformation, validation, storage, and audit history.

## Purpose

Ensures traceability, reproducibility, and trust.

## Fields

See `entities/provenance-record/provenance-record.documentation.md`.

## Relationships

- DESCRIBES → any entity

## Examples

- A HadCRUT5 dataset downloaded, homogenized, validated, and stored.

## Non-Examples

- A user action log.
- A single timestamp without context.

## Validation Rules

- Origin source is mandatory.
- Every event has action, actor, and timestamp.

## Version Rules

- Append-only event log.

## Governance Rules

- Generated by data pipelines.
- Manual edits forbidden.

## Future Evolution Rules

- W3C PROV-O alignment and hash-linked verification.
